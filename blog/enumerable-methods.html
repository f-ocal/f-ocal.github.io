Enumerable

Enumerable is your best friend in Ruby!!

Enumerables provides methods that allows you to transform, manipulate and sort the collection of data. Enumerable is actually a "module", means that it is a bunch of methods packaged together that can be included with other classes (like Array and hash) Programmers don't have to write all those methods many different times, they just write them once, package them as Enumerable, tell Array and Hash to include them!!!

When you are building any programs and websites, Enumerable gives you lots of opportunity of doing something like executing code block to every element of a collection object. Enumeration means that the range is treated as a traversable collection of individual items.



Let's say we stored information about these jeans in the following data structure:
jeans = [ { color: "sky_blue", price:"$15.99" }, { color:"navy_blue", price:"$15.99" }, { color:"blue", price:"$15.99" }, { color:"navy_blue", price:"$15.99" } ]

We can run code and iterate through each element of the array(each pair of jeans) with method .map . We want to modify the jeans by keeping all of them, once we modify them, we'd like to display the results. map is invoked by a collection and accepts a block of code that acts upon each element in that collection. So, .map will run your block of code and give you an object filled with whatever your block returned each time. .collect is the same as .map. Please see some of the famous enumerable methods and their return results as following:

>jeans.map {|hash| hash[:color] == 'navy_blue'} =>[false, true, false, true]
> jeans.select {|hash| hash[:color] == 'navy_blue'} => [{:color=>"navy_blue", :price=>"$15.99"}, {:color=>"navy_blue", :price=>"$15.99"}]
> jeans.each {|hash| hash[:color] == 'navy_blue'} => [{:color=>"sky_blue", :price=>"$15.99"}, {:color=>"navy_blue", :price=>"$15.99"}, {:color=>"blue", :price=>"$15.99"},{:color=>"navy_blue", :price=>"$15.99"}]
> jeans.collect {|hash| hash[:color] == 'navy_blue'} => [false, true, false, true]
